# 缠论研究模块产品计划

## 1. 背景与资料摘要

缠论通常指“缠中说禅”相关技术分析体系。公开资料中，它把市场走势拆解为可递归观察的结构，包括分型、笔、线段、中枢、走势类型、背驰、盘整背驰以及三类买卖点等。广发证券 2012 年量化专题报告也把这些概念列为该理论框架的核心组成，并单独研究了分型通道趋势系统。

从工程视角看，缠论适合被产品化为“结构识别 + 证据展示 + 可复现实验”的研究工具，而不是直接产品化为交易建议。原因有三点：

- 缠论包含多层级、递归和形态确认规则，适合生成结构化解释。
- 分型、笔、线段、中枢等可以被序列化为确定的研究对象，并接入现有扫描器和图表。
- 公开开源实现提醒了一个关键风险：当新增 K 线后，部分“当前帧”识别结果可能变化或消失，所以系统必须区分 `confirmed` 与 `provisional` 状态。

可参考资料：

- [缠中说禅技术原理整理](https://czsc.readthedocs.io/en/0.9.6/_static/%E7%BC%A0%E4%B8%AD%E8%AF%B4%E7%A6%85%E6%8A%80%E6%9C%AF%E5%8E%9F%E7%90%86.html)
- [广发证券：基于缠中说禅之分型通道趋势交易系统](https://www.xcsc.com/upload/jrcpcx/20120315/160745.pdf)
- [waditu/czsc](https://github.com/waditu/czsc)
- [Vespa314/chan.py](https://github.com/Vespa314/chan.py)
- [chan.py quick guide](https://github.com/Vespa314/chan.py/blob/main/quick_guide.md)

## 2. 产品定位

产品名称建议：缠论结构研究模块。

定位：在 AI Quant Research Platform 中增加一个可选研究扩展，帮助用户基于历史 A 股 K 线数据识别、复核和解释缠论结构。它服务于学习、研究、实验复现和 AI 报告辅助，不提供投资建议，不提示交易动作，不连接券商，也不做自动下单。

核心价值：

- 把人工画图中的分型、笔、线段、中枢等结构转成可追溯数据。
- 把缠论信号纳入现有 `signal_definitions` / `technical_signals` 版本化框架。
- 在 K 线图上叠加结构层，让用户看到“为什么识别为这个结构”。
- 为 AI 研究报告提供结构化上下文，但只输出信息性观察。

## 3. 目标用户与核心场景

目标用户：

- 学习缠论的量化研究初学者。
- 希望把主观画线过程标准化的个人研究者。
- 需要比较不同参数和规则版本的技术分析研究者。
- 想把技术结构纳入 AI 研究报告的教育或研究用户。

核心场景：

1. 用户选择一只 A 股，打开 K 线图，勾选“缠论结构”图层，查看分型、笔、线段和中枢。
2. 用户运行扫描器，选择缠论研究信号，如“底分型确认”“中枢突破回抽观察”“背驰观察”，系统记录匹配证据。
3. 用户查看某次扫描结果，打开单只股票详情，看到结构图层、匹配值和中性解释。
4. 用户生成 AI 研究笔记，笔记引用已存储的缠论结构和普通技术信号，输出研究观察与限制说明。
5. 用户上传自己有权处理的缠论学习材料到 RAG 知识库，通过检索理解术语，但不让模型自由编造操作建议。

## 4. 产品边界

明确包含：

- 日线、60 分钟、30 分钟 K 线上的结构识别。
- K 线包含关系处理、顶底分型、笔、线段、中枢、背驰度量和三类买卖点的研究标签。
- `confirmed` 与 `provisional` 状态，展示结构是否可能随新增 K 线变化。
- 版本化规则、参数、算法说明、扫描证据和图表叠加。
- 回测前的数据集冻结和防未来函数检查。
- AI 研究笔记中的信息性解释和限制说明。

明确不包含：

- 真实或模拟下单。
- 券商 API、账户、持仓、订单、资金管理。
- “应该买入”“立即卖出”“推荐排名”等行动导向文案。
- 对未来收益、胜率或价格目标作确定承诺。
- 未经授权抓取付费研报或远程资料。

术语策略：

- 数据库和算法中可以保留 `type_1_buy_point` 等理论术语。
- 前端默认显示为“缠论一类观察点”“缠论二类观察点”等研究标签。
- 解释文案必须加上“仅为历史结构识别，不构成投资建议”。

## 5. 功能规划

### 5.1 缠论结构分析器

新增一个独立分析服务，建议位置为 `scanner/chan/`。它不直接写数据库，而是接收标准化 K 线，输出结构快照。

核心输出：

- `normalized_bars`: 处理包含关系后的 K 线。
- `fractals`: 顶分型、底分型，包含位置、价格、确认状态。
- `strokes`: 笔，包含方向、起止分型、极值、确认状态。
- `segments`: 线段，包含方向、起止笔、确认状态。
- `centers`: 中枢，包含级别、区间上沿、区间下沿、起止结构。
- `divergences`: 背驰观察，包含使用的动量度量，如 MACD 面积或价格力度。
- `research_labels`: 三类买卖点的研究标签，包含理论类型、证据和状态。

第一版应只实现最小闭环：包含处理、分型、笔、中枢和少量研究信号。线段和背驰可以在规则说明稳定后分阶段加入。

### 5.2 版本化信号

复用现有扫描器的规则模型，新增以下信号定义：

- `chan_fractal_confirmation` v1：确认顶分型或底分型。
- `chan_stroke_turn` v1：笔方向发生确认切换。
- `chan_center_formation` v1：形成新的中枢区间。
- `chan_center_leave_retest` v1：价格离开中枢后出现回抽或回试观察。
- `chan_divergence_observation` v1：在已确认结构上出现背驰观察。

每个信号必须写清：

- 所需最少 K 线数量。
- 支持的周期。
- 使用的价格字段和调整方式。
- 是否允许 `provisional` 信号入库。
- 匹配值 JSON 的字段结构。
- 中性解释模板。

### 5.3 数据持久化

MVP 可以先只把缠论信号写入现有 `technical_signals`，把结构明细放入 `matched_values`。当图表叠加需要完整结构历史时，再增加专用表。

建议第二阶段新增：

- `chan_analysis_runs`: 记录股票、周期、算法版本、参数、数据范围、分析时间、是否增量计算。
- `chan_elements`: 记录分型、笔、线段、中枢、背驰、研究标签等统一元素。
- `chan_element_links`: 记录元素之间的父子或组成关系，例如中枢由哪些笔或线段构成。

关键字段：

- `element_type`: `fractal`, `stroke`, `segment`, `center`, `divergence`, `research_label`
- `direction`: `up`, `down`, `neutral`
- `status`: `confirmed`, `provisional`, `invalidated`
- `start_time`, `end_time`, `start_index`, `end_index`
- `price_low`, `price_high`, `evidence`
- `algorithm_code`, `algorithm_version`, `config_hash`

### 5.4 API 设计

短期复用：

- `GET /api/v1/signals?signal_code=chan_*`
- `GET /api/v1/stocks/{symbol}/signals?signal_code=chan_*`
- `POST /api/v1/scanner-runs` 中允许选择缠论信号。

新增只读接口建议：

- `GET /api/v1/stocks/{symbol}/chan-analysis`
- `GET /api/v1/stocks/{symbol}/chan-analysis/{analysis_id}`

查询参数：

- `exchange`
- `interval`: `30m`, `60m`, `1D`
- `from_date`, `to_date`
- `algorithm_version`
- `include_provisional`
- `layers`: `fractals,strokes,segments,centers,divergences,labels`

响应需要包含：

- 数据范围与 K 线数量。
- 算法版本与参数摘要。
- 每个图层的元素列表。
- 结构状态和不确定性说明。
- 与现有 `technical_signals` 的关联 ID。

### 5.5 前端体验

在现有 K 线图上增加图层控制：

- 分型点。
- 笔。
- 线段。
- 中枢区间。
- 背驰观察。
- 研究标签。

交互设计：

- 图表工具栏使用开关或分段控件控制图层。
- Hover 某个元素时显示结构类型、起止时间、价格区间、状态和证据。
- 右侧详情面板显示当前选中结构的组成关系。
- `provisional` 元素用虚线或低对比样式，`confirmed` 元素用实线。
- 所有文案使用“观察”“结构”“证据”，避免“机会”“买入”“卖出”作为行动提示。

### 5.6 AI 与 RAG 集成

AI 报告输入只允许来自存储上下文：

- 近期价格摘要。
- 普通技术信号。
- 缠论结构快照。
- 缠论研究信号。
- RAG 检索到的用户授权材料片段。

新增 prompt 版本：

- `chan-research-note-v1`

报告结构：

- 数据范围。
- 缠论结构观察。
- 与普通技术信号的交叉验证。
- 不确定性与失效条件。
- 研究限制。

安全要求：

- 输出必须标注为信息性研究。
- 不生成操作建议、仓位建议或确定性预测。
- 对 `provisional` 结构必须明确说明可能变化。

## 6. 分期路线图

### Phase C0: 规则文档与夹具

目标：先把可实现的缠论子集写清楚，避免不同理解混进代码。

任务：

- 编写 `docs/chan_theory_rules.md`，定义第一版包含处理、分型、笔、中枢规则。
- 准备 5 到 10 组手工标注 K 线夹具。
- 明确不处理或延后处理的争议规则。
- 评估是否参考开源库实现，但不要在未完成许可证和依赖评估前直接引入。

验收：

- 每条规则都有输入、输出、边界条件和示例。
- 每个夹具都有预期结构和人工说明。
- 文档明确研究用途和非投资建议边界。

### Phase C1: 核心分析器

目标：实现纯函数式结构分析，不接 API、不接数据库。

任务：

- 新增 `scanner/chan/` 数据类型和分析函数。
- 实现包含关系处理、分型识别、笔识别和中枢识别。
- 输出 `confirmed` / `provisional` 状态。
- 添加单元测试、边界测试和快照测试。

验收：

- 同一 K 线输入得到确定、可重复的结构输出。
- 夹具测试全部通过。
- 不引入交易执行、资金管理或券商逻辑。

### Phase C2: 扫描器信号集成

目标：把缠论结构转成现有版本化信号。

任务：

- 新增 `chan_*` signal definitions。
- 在扫描器中加载足够历史 K 线并调用分析器。
- 将研究标签和关键结构证据写入 `technical_signals.matched_values`。
- 添加 PostgreSQL 集成测试和 CLI 测试。

验收：

- 可通过 CLI 运行缠论信号扫描。
- 匹配结果可在现有 `/signals` API 中查询。
- 每个匹配都有算法版本、参数、结构证据和中性解释。

### Phase C3: 结构 API 与图表叠加

目标：让用户能看见结构，而不是只看到信号表。

任务：

- 增加只读 `chan-analysis` API。
- 前端 K 线图增加图层开关和结构 hover 详情。
- 支持日线、60 分钟、30 分钟结构层。
- 保留现有图表性能和移动端可读性。

验收：

- 用户能在图表上切换分型、笔、中枢等图层。
- `provisional` 和 `confirmed` 状态视觉上可区分。
- API 和前端测试覆盖空数据、短历史、歧义 symbol 和后端错误。

### Phase C4: AI 研究笔记增强

目标：让 AI 报告引用缠论结构，但保持可追溯和中性。

任务：

- 扩展研究笔记上下文构造器。
- 新增 `chan-research-note-v1` prompt。
- 加入禁止行动建议的安全测试。
- RAG 检索可返回用户授权的缠论资料片段作为解释参考。

验收：

- 报告能引用缠论结构和信号证据。
- 报告明确数据范围、结构状态和限制。
- 禁止推荐、收益保证和交易动作语言。

## 7. 测试与质量要求

核心测试：

- 包含关系合并顺序测试。
- 顶底分型边界测试。
- 笔最小间隔、同向分型替换和极值选择测试。
- 中枢区间重叠测试。
- `confirmed` / `provisional` 状态随新增 K 线变化的回归测试。
- 防未来函数测试：扫描某日期时只能使用该日期及以前数据。
- 信号持久化唯一性和幂等测试。
- API 分页、过滤、错误响应测试。
- 前端图层渲染、hover、空状态和窄屏测试。
- AI 输出安全门测试。

质量门槛：

- 所有算法规则必须有文档、版本和夹具。
- 任何规则变更必须创建新版本。
- 图表和报告必须展示算法版本和数据范围。
- 默认测试不依赖实时行情、付费服务或外部网络。

## 8. 主要风险与缓解

解释歧义：

- 风险：缠论不同流派对笔、线段、中枢细节理解不同。
- 缓解：产品中声明“本平台规则版本”，所有输出绑定算法版本。

信号重绘：

- 风险：新增 K 线后，当前结构可能变化。
- 缓解：存储 `provisional` / `confirmed` / `invalidated`，并在 UI 中明确展示。

未来函数：

- 风险：回看完整走势会误用未来数据。
- 缓解：扫描器按评估日期截断历史，测试中强制校验。

过度投资化：

- 风险：“买卖点”术语容易被理解为操作指令。
- 缓解：前端和报告统一使用“研究标签”“观察点”，保留安全免责声明。

依赖风险：

- 风险：直接引入开源库会带来许可证、性能、接口稳定性和规则适配问题。
- 缓解：第一阶段自建最小规则闭环；开源库只作为评估参考或可插拔 adapter。

性能风险：

- 风险：多股票、多周期结构计算开销高。
- 缓解：先做批量离线扫描；需要时增加分析缓存和增量更新。

## 9. 推荐 MVP

第一版不要追求“完整缠论”。推荐 MVP 是：

- 日线结构识别。
- 包含处理、分型、笔、中枢。
- 3 个版本化研究信号。
- 现有信号 API 可查询。
- K 线图显示分型、笔和中枢。
- AI 报告能引用这些结构，但只输出中性研究观察。

这个范围足够展示产品价值，也能守住项目“研究和教育”的边界。
